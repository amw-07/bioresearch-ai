# BioResearch AI

**Live demo:** https://bioresearch-ai.vercel.app  
**GitHub:** https://github.com/Irfhan-04/bioresearch-ai  
**Stack:** FastAPI · XGBoost · SHAP · sentence-transformers · ChromaDB · Claude API (`claude-haiku-4-5-20251001`) · Next.js · Render.com · Vercel · Supabase · Upstash Redis

BioResearch AI is a research intelligence app that discovers, ranks, and explains biotech researchers from PubMed using four production-style AI components.

---

## What it does

You can type natural language queries such as:
- `organ-on-chip hepatotoxicity assay`
- `preclinical drug safety ADME`
- `DILI biomarker clinical translation`

The app returns researcher profiles with:
- **Relevance Score** (0–100) from an XGBoost model
- **Semantic Match** (%) from sentence-transformer embeddings
- **SHAP explanation** for feature-level contribution
- **Researcher Intelligence** summary generated by Claude

---

## What makes it an AI project

This app combines four separate AI/ML components at inference time:

1. **XGBoost Relevance Scorer** (`backend/app/services/scoring_service.py`)  
   Structured features are used to predict High / Medium / Low relevance tiers.

2. **Semantic Retrieval** (`backend/app/services/embedding_service.py`)  
   Query and abstract embeddings are stored and searched in ChromaDB with cosine similarity.

3. **LLM Intelligence Generation** (`backend/app/services/intelligence_service.py`)  
   Claude returns structured JSON fields (`research_summary`, `domain_significance`, `key_topics`, `data_gaps`, etc.).

4. **SHAP Explainability** (`backend/app/services/scoring_service.py`)  
   Top SHAP contributions are surfaced in the UI to explain why each score was assigned.

## Model performance snapshot

Metrics are generated by running the training script locally:

```bash
cd backend
uv run python scripts/generate_training_data.py
uv run python ml/train_scorer.py
# Prints macro F1 to terminal and writes backend/ml/reports/eval_v1.json
```

On Render.com, this script runs at Docker build time — `eval_v1.json` is generated during deployment and served at `GET /scoring/metrics`. The `/dashboard/scoring` page reads live values from that endpoint.

Expected macro F1 range: 0.80–0.88 depending on random seed.


## Technical decisions

- **Why all-MiniLM-L6-v2:** compact model size, CPU-friendly latency, good domain synonym behavior.
- **Why ChromaDB:** embedded vector store with minimal infra overhead.
- **Why XGBoost:** strong tabular performance and direct compatibility with exact TreeExplainer SHAP values.
- **Why macro F1:** class-balanced metric for High/Medium/Low tier quality.

---

## What I would do differently

1. **Create expert-labeled training data** instead of relying on synthetic labels.
2. **Use persistent vector storage** for deployment environments with ephemeral disk.
3. **Add async progress streaming** for enrichment jobs in the UI.
4. **Fine-tune embeddings** on biotech-specific contrastive pairs for stronger retrieval quality.

## Local setup

Requirements: Python 3.11+, Node.js 18+, PostgreSQL, Redis.

```bash
# clone
git clone https://github.com/Irfhan-04/bioresearch-ai.git
cd bioresearch-ai

# backend
cd backend
cp .env.example .env
python scripts/generate_training_data.py
# training requires full python deps installed
python ml/train_scorer.py
python scripts/seed_demo_data.py
python -m uvicorn app.main:app --reload

# frontend
cd ../frontend
cp .env.example .env.local
npm install
npm run dev
```
